## üìñ Documentation Standards

All FRIDAY documentation follows these principles:

### **Numbering System**
- **Level 1:** Main category (1, 2, 3, 4, 5)
- **Level 2:** Documents within category (1.1, 1.2, 1.3)
- **Level 3:** Sections within document (1.1.1, 1.1.2)
- **Level 4:** Subsections (1.1.1.1, 1.1.1.2)

### **Structure**
Each document must include:
- ‚úÖ Clear title
- ‚úÖ Version number
- ‚úÖ Last updated date
- ‚úÖ Table of contents (for long docs)
- ‚úÖ Numbered sections
- ‚úÖ Code examples (where applicable)
- ‚úÖ Related documents section

### **Formatting**
- Use British English spelling
- Include emoji for visual clarity
- Code blocks with language specification
- Clear section headers
- Consistent numbering
